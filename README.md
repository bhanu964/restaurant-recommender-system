# Predictive Restaurant Recommendation System

## Overview
This project builds a machine learning system to predict which restaurants a customer is likely to order from. It uses customer data, location information, vendor details, and past order history to generate predictions.

The goal is not just to train a model, but to build a complete pipeline that can take raw data and produce usable recommendations.

---

## Problem
Given a customer and their location, predict the likelihood of ordering from each available vendor.

The output is a score for every (customer, location, vendor) combination.

---

## Approach

### Data Preparation
The data comes from multiple sources:
- Customer data
- Location data
- Vendor data
- Historical orders

Column names were inconsistent across files, so all columns were standardized to lowercase before processing. The datasets were then merged using customer_id and location_number.

---

### Dataset Construction
The model requires both positive and negative examples.

- Positive samples: actual customer orders
- Negative samples: vendors that the customer did not order from

Negative samples were generated by randomly selecting vendors not present in the customer’s order history. This step is important to avoid bias and make the model learn meaningful patterns.

---

### Feature Engineering
The following features were created:

- Distance between customer and vendor (based on latitude and longitude)
- Vendor popularity (number of orders per vendor)
- Customer activity (number of past orders)
- Vendor category
- Location type (home, work, etc.)

These features capture both user behavior and contextual information.

---

### Encoding
Categorical variables were converted using one-hot encoding:
- location_type
- vendor_category_en
- gender

All remaining non-numeric columns were removed before training.

---

### Model
- Algorithm: Random Forest Classifier
- Train/validation split: 80/20
- Evaluation metric: ROC-AUC

Final ROC-AUC score: approximately 0.94

---

## Pipeline

Raw data is processed through the following steps:

Data loading → merging → feature engineering → encoding → model training → prediction → submission generation

Install dependencies:

Train the model:

python src/train_model.py

Generate predictions:

python src/predict.py

---

## Output Format

Each line in the output file follows this format:

CID X LOC_NUM X VENDOR target

Example:

Z59FTQD X 0 X 243 0.13

---

## Challenges

- Inconsistent column names across datasets (uppercase vs lowercase)
- Handling missing values after merging multiple sources
- Generating negative samples without introducing bias
- Ensuring train and test pipelines use the exact same features
- Avoiding row mismatches during prediction

---

## Key Takeaways

- Feature engineering had a bigger impact than model choice
- Data consistency is critical in multi-source systems
- Recommendation problems require careful dataset design
- Building a working pipeline is more important than just training a model

---

## Possible Improvements

- Ranking-based models instead of binary classification
- Top-K recommendations per user
- Gradient boosting models (XGBoost or LightGBM)
- Serving predictions through an API
